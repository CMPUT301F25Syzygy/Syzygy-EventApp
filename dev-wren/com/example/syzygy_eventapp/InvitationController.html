Package com.example.syzygy_eventapp

Class InvitationController

  • All Implemented Interfaces:
    
public class InvitationController

    Controller for reading/writing Invitation data in Firestore DB. Firestore is the source of truth; views have real-time listeners and render Invitation models from snapshots. Writes (create/accept/reject/cancel) go through this controller.

         Task<List<String>> createInvites(String event, String organizerID, List<String> recipientIDs)

        Create one or many invitations for an event in a single Firestore batch write. Each recipient gets its own document. Initial state: accepted == null (pending), sendTime = serverTimestamp(), cancelled = false, cancelTime = null.

        Parameters:
        event - Event document ID
        organizerID - Organizer user ID creating the invitations
        recipientIDs - List of recipient user IDs
        Returns:

        Task resolving to the list of created invitation document IDs
